# グローバルヘッダー デザイン・レイアウト分析と改善提案

## 現状分析

### 現在の構造
```
┌─────────────────────────────────────────────────────────┐
│ [balubo]     [フィード] [ポートフォリオ]    [🔔] [+] [👤] │
└─────────────────────────────────────────────────────────┘
```

### レイアウト構成
- **左側**: ロゴ（`flex-shrink-0`）
- **中央**: ナビゲーションリンク（`flex-1 justify-center`、md以上で表示）
- **右側**: アクション（通知ベル、作品追加ボタン、プロフィールメニュー、`ml-auto`）

### 現在のスタイル
- 高さ: `h-20` (80px)
- 背景: `bg-white/90 backdrop-blur-xl`
- パディング: `px-6 sm:px-8 lg:px-12`
- ボーダー: `border-b border-gray-200/80`
- シャドウ: `shadow-elegant`

## 問題点と改善提案

### 1. モバイルナビゲーション
**問題**: 
- デスクトップのナビゲーションが`hidden md:flex`で非表示
- モバイルでは下部ナビゲーションのみ
- ハンバーガーメニューがない

**改善案**:
- モバイル用のハンバーガーメニューを追加
- または、モバイルでも上部にナビゲーションを表示（アイコンのみ）

### 2. レスポンシブデザイン
**問題**:
- 小さい画面で「作品追加」ボタンのテキストが切れる可能性
- 右側のアクションが多すぎて圧迫感がある

**改善案**:
- モバイルでは「作品追加」ボタンをアイコンのみに
- 通知ベルとプロフィールメニューの間隔を調整

### 3. 視覚的階層
**問題**:
- ナビゲーションリンクのアクティブ状態が分かりにくい
- ドロップダウンメニューの位置が不安定

**改善案**:
- アクティブ状態の視覚的強調を改善
- ドロップダウンメニューの位置を固定

### 4. アクセシビリティ
**問題**:
- キーボードナビゲーションのサポートが不十分
- フォーカス状態の視覚的フィードバックが弱い

**改善案**:
- フォーカスリングの追加
- ARIA属性の追加

### 5. スペーシングとパディング
**問題**:
- 高さが80pxで少し大きい
- パディングが画面サイズによって大きく変わる

**改善案**:
- 高さを`h-16` (64px)に調整
- パディングをより統一

### 6. ドロップダウンメニュー
**問題**:
- `onMouseLeave`で閉じるが、モバイルでは動作しない
- クリックアウトサイドで閉じる機能がない

**改善案**:
- クリックアウトサイド検出を追加
- モバイルでのタッチ操作を改善
