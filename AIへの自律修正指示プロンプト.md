### 🤖 AIへの自律修正指示プロンプト

**あなたは、モダンで親しみやすいUI/UXデザインを専門とする「デザインシステム・マネージャー」です。**

これから、既存のLPデザイン（balubo）の**「デザインの一貫性」**を完璧にするためのリファクタリング（自律修正）を実行してください。

以下の**「統一デザインルール」**をLPの全セクション（ヘッダーからフッターまで）に適用し、このルールから逸脱している箇所を**すべて自律的に検出・修正**してください。

---

### 【統一デザインルール】

---

#### # 1. 全体のトーン

- **テーマ:** 「親しみやすさ」「清潔感」「柔らかさ」を最優先とする。
- **インスピレーション:** 「Nani!?」 (nani.now) のデザインテイスト。

---

#### # 2. 配色ルール（厳守）

- **セクション背景（カード有）:** 情報カードを配置するセクション（例: 特徴、課題、利用者の声、FAQ など）の背景は、すべて **「ごく薄い青」（例: `#F4F7FF`）** に統一する。
- **セクション背景（カード無）:** 上記以外のセクション（例: ヒーロー、フッターなど）は **「白」(`#FFFFFF`)** を基調とする。
- **カード:** すべてのカード型コンポーネント（情報カード、実績カード、FAQアコーディオン、入力フォームなど）の背景は **「真っ白」（`#FFFFFF`）** とし、必要に応じて `bg-white/95` のような軽い透過を使ってもよい。
- **CTAボタン:** すべての主要CTAボタン（「無料で登録する」「無料で始める」など）は、**鮮やかな青（例: `#007AFF`）の背景色＋白文字**で統一する。
- **テキスト:** 見出しと本文の基本色は **黒またはダークグレー（例: `#111111`〜`#333333`）** に統一する。装飾的なアクセント以外でのカラー文字乱用は禁止。

---

#### # 3. 形状と影のルール（最重要）

- **カードの角丸:**
  - すべてのカード型コンポーネントに、**「非常に大きな角丸」**を適用する（目安: `border-radius: 24px` 以上。Tailwind なら `rounded-2xl`〜`rounded-[28px]` 程度）。
  - 中途半端な角丸 (`rounded-md`, `rounded-lg` など) が混在している場合は、上記ルールに合わせて修正する。
- **ボタンの角丸:**
  - すべての主要CTAボタンは、角丸が高さの半分以上になる **「ピル形状」（例: `border-radius: 999px` または `rounded-full`）** に統一する。
- **影（box-shadow）:**
  - すべてのカードと主要ボタンには、**柔らかく、ぼかしの大きい影**を適用する（例: `0 18px 45px rgba(15, 23, 42, 0.06)` や `0 20px 50px rgba(37, 99, 235, 0.35)` など）。
  - 影が濃すぎる、エッジが固すぎる、あるいは方向がバラバラな箇所は、上記トーンに合わせて修正する。

---

#### # 4. アイコンのルール

- LP全体で使用されるすべてのアイコン（特徴、ステップ、チェックマーク、プロフィールなど）は、**「クレイモーフィズム（ソフト3D）」風のスタイル**に統一する。
  - 典型的なスタイルは、「外側の淡い丸いContainer＋内側の小さめのグラデーション丸＋柔らかい影」構造。
- フラットな線画アイコンのみで構成されている箇所は、必要に応じて上記のクレイ風コンテナに包み、LP全体のアイコンとテイストを揃える。

---

#### # 5. レイアウトと整列のルール

- **情報セクションのテキスト整列:**
  - 見出し（H2）とその直下の説明文を持つすべての情報セクションでは、**原則としてテキストを「左揃え」に統一**する。
  - 例: 「こんなお悩みはありませんか？」「baluboが可視化する3つの価値」「よくあるご質問」など。
  - 現在「中央揃え」になっている情報見出し・本文があれば、すべて左揃えに修正する。
- **CTAセクションの整列:**
  - 「最終的な行動喚起」を目的とするセクション（例: 中間CTA、Final CTA など）は、**中央揃えを維持してよい**。ただし、見出し・本文・ボタンの横位置は一貫して中央に揃える。
- **カード内テキストの開始位置:**
  - 複数カラムのカード（2列・3列など）が並ぶ場合、各カード内の「タイトル＋本文」ブロックの上端が **同じY座標に揃う** ように余白・`min-height`・Flexレイアウトを調整する。
  - タイトルの行数が異なるカードが混在する場合は、`min-h` や余白を使って「本文開始位置」が水平にそろうようにする。

---

#### # 6. ノイズ削除のルール

- **不要な英語ラベルの削除:**
  - デザイン上の意味を持たない英語ラベル（例: `"Why balubo?"`, `"Voices"`, `"Recently Added"`, `"Final step"`, `"FAQ"` など）は、すべて削除する。
  - 代わりに、日本語のメイン見出し（H2）がセクションのタイトルとして機能するようにする。
- **冗長な小見出しの削除・簡略化:**
  - 「専門性可視化までの3ステップ」など、見れば分かる情報を重ねて説明しているサブ見出しは削除または簡略化する。
  - 目的は「ミニマルで読みやすい」こと。意味が重複するテキストや装飾的なラベルは可能な限り削る。

---

#### # 7. セクション区切り（トランジション）ルール（新規追加）

- **直線の境界線は禁止し、波形デバイダーを採用する。**
  - セクション間の切り替えは、単純な直線の境界ではなく、添付デザイン例にあるような **「柔らかい波形（wave）」のSVGデバイダー** を使用する。
- **適用ルール:**
  - 「白背景 → 薄い青背景」に切り替わる箇所では、**上側（白セクション）の最下部**に「薄い青」の波形デバイダーを配置する。
  - 「薄い青背景 → 白背景」に切り替わる箇所では、**上側（薄い青セクション）の最下部**に「白」の波形デバイダーを配置する。
  - これにより、背景色の切り替えが直線ではなく、常に柔らかい曲線でつながるようにする。
- **実装上の注意:**
  - 波形は再利用可能なコンポーネント（例: `<SectionDivider colorClass="text-blue-50" heightClass="h-10" />` のようなコンポーネント）として実装し、すべてのセクション切り替えで一貫して使う。
  - セクション間で余白が二重にならないよう、`margin` / `padding` とデバイダーの高さを考慮してレイアウトを調整する。

---

### 実行指示

上記 **7つの統一デザインルール** に基づき、現在の balubo LP の HTML / CSS / コンポーネント構造を分析し、  
ヘッダーからフッターまでの**すべてのセクション**に対して以下を実行してください。

1. 各セクション・コンポーネントが、ルール #1〜#7 を満たしているか自動的にチェックする。
2. ルールに反している箇所を自律的に特定し、デザインとコードを**一貫性のある状態に修正**する。
3. 修正後のコード（またはコンポーネント定義）を、元の構造を壊さない範囲で出力する。
4. 必要に応じて、変更したポイントと理由を簡潔にコメントとしてコード内に残す。

このプロンプトに基づき、あなた自身の判断で最適なデザイン・実装を行い、  
LP全体を「親しみやすく、清潔感があり、柔らかい」トーンで**完全に統一された状態**にリファクタリングしてください。

## ✍️ クリエイター分析: Signature Styles 抽出プロンプト

ライター／編集者のポートフォリオ上で、「作業内容」ではなく「提供価値」として強みを見せるための生成指示です。  
AIには以下の観点を必ず守らせてください。

### 目的
- 発注者（編集プロダクション／企業Web担当者）が「この人に頼みたい」と思えるプロのスキルタグを作る。
- スタイル＝武器として表現し、「どんな案件に刺さるのか」が一目で分かる構成にする。

### 指示方針
1. **価値で語る:** 「厚労省の通知を分析した」→「難解な専門情報の翻訳と大衆化」のようにリライト。
2. **業界用語を使う:** 取材力・構成力・トーン＆マナー・ストーリーテリング・ファクトチェック等。
3. **利用シーンを想起:** 「BtoBマーケ資料」「採用広報」「ホワイトペーパー」といった案件を想像できる表現。

### 入力例
- Cloudflare機能を比較表や図解で紹介
- 厚労省通知を分析し対象薬剤を可視化
- 専門用語を噛み砕き、読者理解を深めた記事

### 出力フォーマット（カード ×4）
```
スタイル名（20文字以内／編集用語を含める）
提供価値: 読者・クライアントへのメリット（約60文字）
活用シーン: 想定案件（ホワイトペーパー等）
根拠（実績）: どの作品から導いたか
ハッシュタグ: #取材 #図解 #ストーリーテリング など3個目安
```

### コピペ用プロンプト
```
現在、AIが分析したライターの過去記事データ（構成や切り口の要約）があります。
これを、発注者が見たときに「この人に仕事を頼みたい」と思える
Signature Styles（編集スキルタグ）として再定義してください。

【変更の方向性】
- 「行政・制度の構造化翻訳」といった硬い表現を、ライター/編集者の武器として魅力的に表現し直す
- 作業ベースではなく「どんな価値を提供できるか」「どんな案件に刺さるか」で語る
- 取材力 / 構成力 / トーン＆マナー / ストーリーテリング / ファクトチェックなど業界用語を含める

【入力データ（例）】
1. Cloudflareの機能を比較表や図解で説明した記事
2. 厚労省の通知を詳細に分析し、対象薬剤を明確化した記事
3. 専門用語を分かりやすく噛み砕き、読者の理解を深めた記事

【出力要件】
- 4つのスタイルを定義し、それぞれ以下を必ず含める
  - スタイル名（20文字以内のキャッチコピー）
  - 提供価値（読者・クライアントメリットを60文字前後で）
  - 活用シーン（ホワイトペーパー/採用広報/カンファレンスレポート等）
  - 根拠（どの実績・構成要素から抽出したか）
  - ハッシュタグ（#取材 #図解 #ストーリーテリング など3つ）

【トーン＆マナー】
- 知的で信頼感がありつつ、クリエイティブ
- 「厚労省の記事を書きました」ではなく「高度な専門情報の翻訳と大衆化」のように価値で語る
```

このプロンプトを `CreatorAnalyticsOverview` のテキスト生成やCMS管理に活用し、UIとは別に内容の品質を担保してください。
