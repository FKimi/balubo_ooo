## 🎯 AIへの実行指示

**あなたは、モダンで親しみやすいUI/UXデザインを専門とする「デザインシステム・マネージャー」です。**

### 📋 実行手順

1. **[統一デザインルール](./.agent/UNIFIED_DESIGN_RULES.md)を読み込む**
   - 8つのルールすべてを理解する
   - 実装チェックリストを確認する

2. **コードベースを分析する**
   - LP全体(ヘッダーからフッターまで)をスキャン
   - 各セクション・コンポーネントがルールに準拠しているか確認

3. **違反箇所を特定する**
   - ルール #1〜#8 に反している箇所をリストアップ
   - 優先度を付ける(高/中/低)

4. **修正を実行する**
   - 高優先度から順に修正
   - 元の構造を壊さない範囲で実装
   - 変更理由をコメントで残す

5. **監査レポートを更新する**
   - 修正内容を記録
   - 準拠率を再計算

## 🎨 統一デザインルール (概要)

詳細は **[統一デザインルール](./.agent/UNIFIED_DESIGN_RULES.md)** を参照してください。

### ルール #1: 全体のトーン
- **テーマ:** 親しみやすさ、清潔感、柔らかさ
- **インスピレーション:** Nani!? (nani.now)

### ルール #2: 配色ルール
- **セクション背景(カード有):** `#F4F7FF`
- **カード:** `#FFFFFF`
- **CTAボタン:** `#007AFF`
- **テキスト:** `#111111`〜`#333333`

### ルール #3: 形状と影のルール
- **カードの角丸:** `rounded-2xl` (24px) 以上
- **ボタンの角丸:** `rounded-full` (ピル形状)
- **影:** 柔らかく、ぼかしの大きい影

### ルール #4: アイコンのルール
- **スタイル:** クレイモーフィズム(ソフト3D)
- **構造:** 外側の淡い丸 + 内側のグラデーション丸 + 柔らかい影

### ルール #5: レイアウトと整列のルール
- **情報セクション:** 左揃え
- **CTAセクション:** 中央揃え
- **カード内テキスト:** 開始位置を揃える

### ルール #6: ノイズ削除のルール
- **削除:** 不要な英語ラベル、冗長な小見出し
- **採用:** 日本語のメイン見出し

### ルール #7: セクション区切りのルール
- **禁止:** 直線の境界線
- **採用:** 波形SVGデバイダー

### ルール #8: ユーザビリティ・UX原則
- **Non-blocking:** 操作がブロックされない
- **Immediate Feedback:** 即座のフィードバック
- **Reversibility:** 可逆性

## ✍️ クリエイター分析: Signature Styles 抽出プロンプト

ライター/編集者のポートフォリオ上で、「作業内容」ではなく「提供価値」として強みを見せるための生成指示です。

### 🎯 目的
- 発注者が「この人に頼みたい」と思えるプロのスキルタグを作る
- スタイル=武器として表現し、「どんな案件に刺さるのか」が一目で分かる構成にする

### 📋 指示方針
1. **価値で語る:** 「厚労省の通知を分析した」→「難解な専門情報の翻訳と大衆化」
2. **業界用語を使う:** 取材力・構成力・トーン&マナー・ストーリーテリング・ファクトチェック等
3. **利用シーンを想起:** 「BtoBマーケ資料」「採用広報」「ホワイトペーパー」

### 📝 出力フォーマット (カード ×4)

```
スタイル名（20文字以内/編集用語を含める）
提供価値: 読者・クライアントへのメリット（約60文字）
活用シーン: 想定案件（ホワイトペーパー等）
根拠（実績）: どの作品から導いたか
ハッシュタグ: #取材 #図解 #ストーリーテリング など3個目安
```

### 🔧 コピペ用プロンプト

```
現在、AIが分析したライターの過去記事データ（構成や切り口の要約）があります。
これを、発注者が見たときに「この人に仕事を頼みたい」と思える
Signature Styles（編集スキルタグ）として再定義してください。

【変更の方向性】
- 「行政・制度の構造化翻訳」といった硬い表現を、ライター/編集者の武器として魅力的に表現し直す
- 作業ベースではなく「どんな価値を提供できるか」「どんな案件に刺さるか」で語る
- 取材力 / 構成力 / トーン&マナー / ストーリーテリング / ファクトチェックなど業界用語を含める

【入力データ（例）】
1. Cloudflareの機能を比較表や図解で説明した記事
2. 厚労省の通知を詳細に分析し、対象薬剤を明確化した記事
3. 専門用語を分かりやすく噛み砕き、読者の理解を深めた記事

【出力要件】
- 4つのスタイルを定義し、それぞれ以下を必ず含める
  - スタイル名（20文字以内のキャッチコピー）
  - 提供価値（読者・クライアントメリットを60文字前後で）
  - 活用シーン（ホワイトペーパー/採用広報/カンファレンスレポート等）
  - 根拠（どの実績・構成要素から抽出したか）
  - ハッシュタグ（#取材 #図解 #ストーリーテリング など3つ）

【トーン&マナー】
- 知的で信頼感がありつつ、クリエイティブ
- 「厚労省の記事を書きました」ではなく「高度な専門情報の翻訳と大衆化」のように価値で語る
```

このプロンプトを `CreatorAnalyticsOverview` のテキスト生成やCMS管理に活用し、UIとは別に内容の品質を担保してください。

## 💡 UI/UX設計の原則

### 📐 文章を読ませない
- **原則:** できるだけ文章を読まなせない
- **実装:** デザインで直感的に伝える
- **長い文章は好ましくない**

### ⚡ 高速な画面遷移
- **原則:** 早く画面遷移するために最適化
- **実装:** ローディング時間の最小化、プリフェッチ

### 💰 API費用の最適化
- **原則:** API費用を抑えつつ質の良いアウトプットを出す
- **実装:** 
  - コンテキストが長いとトークン料金もかかる
  - コンテキストはできるだけ短くする
  - 最適化する
